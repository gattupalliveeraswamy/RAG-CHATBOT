In today’s AI-driven world, the ability to retrieve accurate, context-rich answers from large collections of documents is a game-changer. This page details an Agent-powered RAG (Retrieval-Augmented Generation) workflow built in n8n to extract specific details from Google Drive PDFs and provide precise, context-aware responses—automating repetitive, time-consuming, and error-prone document tasks.

Problem Statement

  • Slow retrieval due to manual file opening and keyword searching.
  • Missed context when using traditional search tools.
  • Inability to scale as document counts increase.

The challenge was to build a system that:

  1. Ingests both existing and new PDFs from Google Drive.
  2. Extracts their content and stores it in a vector database.
  3. Responds to user queries with accurate, context-aware answers.

Approach & Methodology

The solution applies Retrieval-Augmented Generation (RAG):

  1. Retrieval — Search a vector database for relevant content.
  2. Augmentation — Feed the retrieved context to an AI model.
  3. Generation — Produce natural-language answers.

Two interconnected workflows power the system:

  1. Data Ingestion — Ingest PDFs, extract content, split text, generate embeddings, and store them in a vector database (bulk & incremental).
  2. Query Processing — Accept user queries, retrieve relevant chunks, and generate answers.

Tools, Frameworks, and Data Used

  • n8n — Automation workflow platform.
  • Google Drive Trigger & Search — Detect and fetch files.
  • PDF Parser — Extract text from PDFs.
  • Recursive Character Text Splitter — Split content into manageable chunks.
  • OpenAI Embeddings API — Create vector embeddings.
  • Pinecone Vector Store — Store and retrieve vectors.
  • OpenAI Chat Model (GPT) — Generate responses with retrieved context.
  • Data Source — PDFs stored in Google Drive.

Workflow Overview

  1. Monitor Google Drive for new or updated PDFs.
  2. Extract and process text content.
  3. Convert text into vector embeddings for semantic search.
  4. Store embeddings in a RAG vector database.
  5. Use an AI Agent to retrieve context and answer questions.

Implementation Steps

Workflow A – Bulk Ingestion (All Existing Files)

  1. Trigger — Manual or scheduled execution.
  2. Search Files & Folders — Retrieve all PDFs from Google Drive.
  3. Download Content — Fetch each file’s content.
  4. Text Splitting — Split documents into chunks with a Recursive Character Text Splitter.
  5. Embedding Generation — Create embeddings via OpenAI.
  6. Vector Storage — Save vectors & metadata in Pinecone.

Workflow B – Incremental Ingestion (New Files Only)

  1. Trigger — Google Drive fileCreated event.
  2. Download File — Retrieve the new file’s content.
  3. Text Splitting — Process content into chunks.
  4. Embedding Generation — Create embeddings for the chunks.
  5. Vector Storage — Add embeddings & metadata to Pinecone without reprocessing existing files.

Query Workflow

  1. Chat Message Received — User sends a query.
  2. AI Agent — Processes the query and decides retrieval strategy.
  3. Vector Store Tool — Searches Pinecone for relevant chunks.
  4. Context Injection — Feeds results into the AI chat model.
  5. Response — Returns an accurate, context-aware answer.

Results and Outcomes

  • Automated Processing — Old and new files handled without manual effort.
  • Fast Query Responses — Relevant answers delivered in seconds.
  • Improved Accuracy — Context-aware retrieval ensures better answers.
  • Scalability — Supports large and growing document sets.

Future Improvements / Next Steps

  • Support more file types (Word, Excel) and database connections.
  • Add triggers for file updates to keep content fresh.
  • Integrate the chatbot into a web app for easier access.
  • Refine retrieval accuracy and embedding performance.
